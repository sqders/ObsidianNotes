# Определение
Injections - изменение стандартного запроса к БД невалидными или другими данными, которые ломают логику взаимодействия с БД, могут способствовать утечке данных. 
## Например
Мы принимает **url-строку**, которая содержит параметр `id=<какой-то id>`. И мы вставляем всё, что после id= в запрос к БД, это плохая практика, т.к. без валидации, пользователь может передать что-то типа: `url.ru?id = 1; DROP TABLE users;` - это и есть инъекция, с помощью которой удалится вообще вся информация о пользователях, если не валидировать строку. 
# Защита от инъекций
Защитой от инъекций в JDBC являются [[Основные элементы JDBC Statements, Connection#PreparedStatements|Prepared Statements]]
