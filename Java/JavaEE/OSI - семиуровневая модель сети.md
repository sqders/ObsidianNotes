**OSI - (Open Systems Interconnection model)** - сетевая модель стека сетевых протоколов OSI/ISO. Посредством данной модели сетевые устройства могут взаимодействовать друг с другом. *Модель определяет уровни взаимодействия систем, которые выполняют свои функции*.
# Уровни
1. Физический - самый низкий уровень, *ближе всего к железу*. **Передача данных-сигналов** физическим образом (провода, устройства, концентраторы передают сигнал и т.п.). Протоколы: **IEEE**(bluetooth, wifi), **IRDA** (передача данных с помощью инфракрасных портов) - описывают, *как работают порты*, например. Физ. уровень не пишут на Java, мб C/C++. Пример: **сетевая карта** - физ.уровень.
2. Канальный уровень - **уровень связи данных**, состоит из 2 подуровней: *физического*(**MAC**) и *аппаратного*. Тут происходит описание того, как передавать программные данные аппаратным образом (с помощью сигналов). Это происходит с помощью **кадра**. Бинарную информацию с компьютера преобразовывают в **кадры** - которые потом передают с помощью протоколов, которые регулируют доставку, целостность. Протоколы: **Ethernet**(протокол организации современных сетей), **Wireless LAN**. Пример оборудования: **драйвера сетевой карты**, передачу данных с неё регулирует канальный уровень. На Java можно писать для канального уровня
3. Сетевой уровень(Network layer) - определение путей передачи данных. НА этом уровне используется Java, конкретный пример - **[[сокеты]]**. Протоколы: IPV4, IPV6, IP. Задача - **куда и как передавать данные**. **При помощи ip-адреса мы определяем, кому и куда передавать информацию**, **при помощи порта мы определяем, как передавать информацию** на аппаратном уровне(канальный и физический уровень), а программная информация передается по средствам [[Сокеты|сокета]].
4. Транспортный  уровень - обеспечивает передачу данных
5. Уровень сеанса (Session layer) - обеспечивает длительную, бесперебойную передачу данных между компьютерами.
6. Уровень представления (Presentation layer)- уровень описания форматов передаваемых данных. Например, **графика** - чтобы передать изображение (большое и мб на плохом соединении), нужен такой формат хранения изображения, чтобы файл загружался **частично**, а не целиком, **послойно**. Пример такого формата - **jpeg, tif**. Однако, тут дело не в частичности, а в **форматах**. Ещё форматы: **avi, mpeg**
7. Прикладной уровень (Application layer) - на этом уровне мы взаимодействуем на уровне программ, браузеров и т.д. Это протоколы HTTP, FTP, SMTP, почтовые, файловые протоколы и т.д.
